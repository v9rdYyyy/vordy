# Atletico Famq AFK Bot

- Slash-команда `/hello` только для администраторов сервера.
- Публикация AFK-панели в текущем канале.
- Кнопка **Встать АФК** открывает форму с 2 полями:
  - причина ухода в АФК
  - примерное время выхода из АФК
- После отправки формы бот обновляет панель и показывает:
  - ник пользователя
  - причину
  - примерное время выхода
  - когда он встал в АФК
  - сколько уже сидит в АФК
- Кнопка **Выйти с АФК** снимает пользователя с AFK-списка.
- Кнопка **Управление** даёт администратору выбрать человека из текущего AFK-списка и снять его с АФК принудительно.
- Данные хранятся в `SQLite`, поэтому список не теряется после перезапуска.
- Кнопки переживают перезапуск бота.

## Структура 

- `main.py` — точка запуска
- `afk_bot/config.py` — загрузка переменных окружения
- `afk_bot/storage.py` — база данных SQLite
- `afk_bot/panel.py` — сборка embed-панели
- `afk_bot/bot.py` — slash-команда, кнопки, модалки, логика бота
- `.env.example` — пример настроек
- `requirements.txt` — зависимости

## Как запустить


```bash
python main.py
```

## пользование

1. Зайди на сервер под администратором.
2. В нужном текстовом канале введи `/hello`.
3. Бот отправит AFK-панель.
4. Пользователи смогут нажимать:
   - **Встать АФК**
   - **Выйти с АФК**
5. Администратор сможет нажать **Управление** и снять любого человека с АФК через выпадающий список.

## Важно про ограничения Discord

Я сделал логику максимально близко к твоему описанию, но у Discord есть 2 важных ограничения:

1. **Кнопки у публичного сообщения общие для всех, кто видит это сообщение.**
   Поэтому кнопку **Управление** нельзя физически скрыть только от обычных пользователей в рамках одного и того же публичного сообщения.
   В этом проекте кнопка видна всем, но срабатывает только у администратора.

2. **Стандартные модальные окна Discord не позволяют вставить кастомный warning-блок внутри формы так же, как на твоём скриншоте.**
   Поэтому форма сделана максимально близко по смыслу: заголовок и 2 текстовых поля.

## Если команда `/hello` не появилась

Обычно причины такие:

- бот не был приглашён со scope `applications.commands`
- не заполнен или неверно заполнен `DISCORD_GUILD_ID`
- команда синхронизируется глобально и Discord ещё не успел её обновить

Если указан `DISCORD_GUILD_ID`, команда обычно появляется быстрее.

## Хранение данных

По умолчанию база создаётся здесь:

```text
./data/afk_bot.sqlite3
```

Можно изменить путь через `.env`:

```env
AFK_BOT_DB_PATH=/полный/путь/до/файла.sqlite3
```
